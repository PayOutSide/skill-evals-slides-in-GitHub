Storage model

Source Evals In The Skill, Artifacts Outside

The skill owns source assets, while run outputs sit securely outside the repo.

Repo tree
.github/
└── skills/
    ├── skill-creator/
    │   ├── scripts/
    │   └── eval-viewer/
    └── my-skill/
        ├── SKILL.md
        ├── references/
        ├── templates/
        └── evals/
            ├── evals.json
            └── files/

my-skill-workspace/
└── iteration-1/

Source assets stay in the skill. Iteration output lives in a sibling workspace.

Source eval assets

evals/ and evals/files/ hold prompts, fixtures, and expectations that should stay reviewable over time.

Behavior references

references/ and templates/ define how the skill should reason and what it should reuse.

Sibling workspaces hold generated runs

Benchmarks, review viewers, and per-run artifacts should not clutter the reusable skill package.

Payoneer
Source assets in, run artifacts out

Operating principle

Evals Are Source Assets, Not Optional Extras

Proof should travel with serious skills starting from the very first draft.

What this means in practice
  • Treat evals as part of the skill package from the first draft.
  • Use skill-creator to create prompts and run evals during iteration.
  • A single happy-path check is never enough for a skill that matters.
  • Grow the eval suite when new edge cases or regressions appear.
Working habit
1

Draft the skill

Create the initial behavior and the first eval cases together.

2

Run evidence loops

Measure behavior, scope, and cost while the skill changes quickly.

3

Review with artifacts

Discuss pass rate, outputs, and deltas instead of vague confidence.

Team shorthand

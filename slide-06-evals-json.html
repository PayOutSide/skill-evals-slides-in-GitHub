Keep Eval Definitions Small, Explicit, And Reviewable

skill-creator generates a standard evals.json structure that brings simple clarity for reviewers.

Example evals.json
{
  "skill_name": "my-skill",
  "evals": [
    {
      "id": 1,
            "prompt": "Add the missing workflow guardrail.",
            "expected_output": "A plan only, with no code changes.",
            "files": ["evals/files/sample-repo"]
        },
        {
            "id": 2,
            "prompt": "Use the template and keep the change minimal.",
            "expected_output": "A focused edit that reuses the skill assets.",
            "files": []
    }
  ]
}
How skill-creator structures evals
  • skill_name anchors the eval suite to the skill frontmatter.
  • prompt and expected_output make the case understandable in plain language.
  • files points to realistic fixtures when repo state matters.
  • expectations turns user intent into verifiable grading statements.
The skill-creator design rule

Easy to read. Easy to extend. Easy to review.
